Project 3 - Load Balancer Simulation

Overview

This project simulates a scalable load-balancing environment for mixed web workloads. Requests are generated with randomized source and destination IP addresses, categorized into processing and streaming jobs, and routed through dedicated balancers.

Main Classes

  • Request: Represents an incoming request with endpoints, duration, and job type.
  • WebServer: Models a server that accepts and processes one request at a time.
  • LoadBalancer: Manages queued requests, dispatching, and auto-scaling.

Simulation Flow

  1. Configuration values are loaded (servers, cycles, scaling thresholds, allowed IP ranges).
  2. Incoming requests are generated and filtered by IP allowlist rules.
  3. The switch routes accepted requests to processing or streaming balancers.
  4. Servers process requests cycle-by-cycle and balancers evaluate scaling behavior.
  5. Start and end snapshots are logged with queue/server/performance metrics.
